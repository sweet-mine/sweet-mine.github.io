Edge AI Vision

Edge AI를 위한 이미지 분류 최적화 (Personal Project)
2025.05 - 2025.06 🔗 상세 노션 링크

개요

Jetson Orin Nano 기반 엣지 디바이스에서 높은 리소스 효율과 성능을 낼 수 있도록 MobileNetV4 기반의 초경량화 비전 분류 모델을 최적화한 프로젝트입니다.

기여한 부분 및 성과

  • 역할: Tiny ImageNet 원본 데이터셋을 활용해 PyTorch로 모델을 훈련시키고, TensorRT 기반으로 높은 추론속도와 정확도를 이끌어냈습니다.
  • 주요 기능: Depthwise Separable Convolution 기반의 자체 모델 아키텍처를 설계하여 정확도를 향상시켰으며, MobileNetV4를 주어진 데이터셋에 맞게 Full-Finetune하여 다양한 해상도의 이미지를 효과적으로 학습시켰습니다.
  • 성과 및 배운점: Jetson Orin Nano 환경에서 정확도 74.32%, 추론 속도 1.87ms를 달성했으며, 전체 파인튜닝 과정에서 학습률의 미세한 차이가 QAT에 미치는 영향을 심도 있게 경험했습니다.

트러블 슈팅 (문제 해결 경험)

1. 양자화(QAT) 과정 중 정확도 하락 및 발산:
- 상황: 초기 높은 학습률로 인해 QAT(Quantization Aware Training) 환경에서 모델이 수렴하지 못하고 정확도가 크게 떨어졌습니다.
- 해결: AdamW 옵티마이저로 오버피팅을 방지하고, Cosine Annealing 스케줄러를 적용하여 세밀하게 가중치를 업데이트한 결과 INT8 양자화 상황에서도 정확도를 2%p 향상시켰습니다.

2. 딥 네크워크 기울기 소실 문제 파악:
- 상황: 모델의 레이어(블록 13개)를 깊게 쌓았을 때 되려 인식 성능이 하락하는 문제를 파악했습니다.
- 해결: Residual Connection(잔차 연결)을 추가 적용하여 Gradient Vanishing을 예방하고, 실험을 거쳐 성능 하락이 방어되는 최적의 블록 수(8개)로 구조를 최적화했습니다.

사용 기술

PyTorch TensorRT QAT MobileNetV4

Socket Chat

소켓 기반 크로스 플랫폼 채팅 서버 (Personal Project)
2025.06 - 2025.06 🔗 상세 노션 링크

개요

운영체제 중심의 통신 구조, 시스템 레벨 네트워크 지식, 소켓 프로그래밍 이해도를 증명하기 위해 기초적인 통신 프로토콜 단계부터 개발한 멀티 플랫폼 채팅 어플리케이션입니다.

기여한 부분 및 성과

  • 역할: 추상화된 C/C++ 표준 네트워크 레이어를 설계해 Windows와 Linux 환경 간 상호 연동이 제한 없이 작동하는 크로스 플랫폼 네트워크 통신을 자체 구현했습니다.
  • 주요 기능: epoll 기반의 Event-Driven 아키텍처를 사용하여 기존 select 서버에 비해 훨씬 빠른 성능을 확보하고, 고정 길이 헤더 + 가변 길이 메시지 방식을 채택하여 효율적인 프로토콜을 설계했습니다.
  • 성과 및 배운점: 서로 다른 OS에서도 원활히 통신 가능한 통신 서버를 성공적으로 구축하였으며, 문자열 인코딩으로 인해 발생할 수 있는 문제점을 깊이 있게 학습했습니다.

트러블 슈팅 (문제 해결 경험)

1. 이기종 OS 간 인코딩 호환성 문제:
- 상황: Linux(Ubuntu)는 기본 시스템 환경에서 UTF-8을 사용하지만, Windows는 CP949 인코딩을 사용하여 상호 통신 시 한글이 심각하게 깨지는 현상이 발생했습니다.
- 해결: 서버 측에서 모든 메시지를 UTF-8로 통일시켜 관리하되, 클라이언트 애플리케이션 수신/송신단(Windows 클라이언트 측)에서만 WideCharToMultiByte 등의 윈도우 API를 사용해 문자열 인코딩 변환 처리를 수행하도록 구조를 개편했습니다. 이를 통해 서버의 플랫폼 독립성과 확장성을 완벽히 보장했습니다.

사용 기술

C / C++ Socket IP/TCP OS / Network Linux

rPPG Health AI

비접촉식 심박수 계산 프로그램 (Team Project) | 3명
2024.08 - 2025.05 🔗 상세 노션 링크

개요

웹캠 영상만으로 사용자의 안면 혈류 변화를 분석하여 심박수를 측정하는 비접촉식 AI 솔루션입니다.

기여한 부분 및 성과

  • 역할: 데이터셋 생성을 위한 데이터 수집 파이프라인(Mediapipe & OpenCV 활용) 프로그램을 자체 개발하고, 1D-CNN과 GRU 결합 모델을 기반으로 한 추론 엔진을 설계 및 구현했습니다.
  • 주요 기능: RGB 채널 색상 혼합 후 데이터셋 생성을 위해 논문을 리서칭하여 적절한 방법을 탐색했고, CV 처리를 위해 여러 라이브러리를 분석하여 Mediapipe를 선택하여 ROI 영역을 안정적으로 추출했습니다.
  • 성과 및 배운점: 실험 참여자 3명의 심박수를 -2~+2 오차범위 내에서 90% 이상의 정확도로 측정하는 성과를 보였으며, 데이터셋이 AI 학습에 얼마나 큰 영향을 미치는지 파악했습니다.

트러블 슈팅 (문제 해결 경험)

1. 얼굴 인식 지연 문제 해결:
- 상황: OpenCV에 기본적으로 내장된 HaarCascade로는 30fps 웹캠의 성능을 이끌어내지 못하고 병목이 발생하는 문제가 생겼습니다.
- 해결: 여러 라이브러리를 사용해보고 성능을 비교하여 Mediapipe를 채택 후 웹캠의 성능을 이끌어내어 문제를 해결했습니다.

2. VRAM 부족(OOM) 문제 해결:
- 상황: 고해상도 영상을 그대로 메모리에 올리면서 GPU Out-Of-Memory 오류가 빈번히 발생했습니다.
- 해결: 얼굴 영역 검출(Cropping) 파이프라인을 추가하여 입력 이미지 사이즈 자체를 축소시키고, 배치(Batch) 조절을 통해 제한된 리소스 내에서 원활하게 학습시켰습니다.

사용 기술

Python PyTorch Mediapipe OpenCV CNN / GRU

ShootDoori

실력 맞춤 지역 스포츠 매칭 플랫폼 (Team Project) | 6명
2025.08 - 2025.11 🔗 상세 노션 링크

개요

지역 기반으로 스포츠 팀과 매치를 생성하고 용병을 모집하며, 실력 기반 리뷰를 남길 수 있는 모바일 플랫폼 서비스입니다.

기여한 부분 및 성과

  • 역할: Spring Boot를 활용한 백엔드 RESTful API 개발 및 데이터베이스 설계를 주도했으며, 핵심 기능인 '매칭 시스템'과 '팀/용병 모집 사이클' 로직을 아키텍처 관점에서 구성했습니다.
  • 주요 기능: 3-Layer-Architecture로 플랫폼을 설계하여 유지보수가 원활하도록 개발하였고, ERD 중심으로 설계하여 복잡해졌던 경기 매칭 로직을 객체 중심으로 재설계했습니다.
  • 성과 및 배운점: 구글 플레이스토어에 앱을 성공적으로 런칭하였으며, 복잡한 연관관계를 가진 데이터베이스를 JPA와 같은 ORM을 활용하여 객체로서 관리하는 방법을 체득했습니다.

트러블 슈팅 (문제 해결 경험)

1. 객체지향적 설계 개선:
- 상황: Service Layer에서 도메인 객체(Team)의 상태를 직접 검사하는 방식 때문에 코드 결합도가 매우 상승해 있었습니다.
- 해결: 상태 검증 로직을 해당 메인 객체 내부로 캡슐화(메서드 위임)하여 결합도를 대폭 낮췄습니다.
- 결과: 요구사항 변경 시 유지보수 및 수정에 소요되는 시간을 절반 수준으로 단축했습니다.

2. 복잡한 매칭 시스템 설계:
- 상황: 다수의 사용자가 동시에 매칭을 요청하거나 팀 신청을 할 때 상태 관리가 복잡하여 충돌 문제가 발생했습니다.
- 해결: 팀 신청, 매칭 완료, 용병 모집 등 복잡한 상태 흐름을 ERD를 작성하며 설계하여 문제를 해결하였습니다.
